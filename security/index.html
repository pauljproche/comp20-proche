Introduction

I am perform a security assessmenting of a potentially vulnerable system. Specifically, I was hired as a security consultant to resolve privacy and security issues relating to a ride-hailing service. That is, I was hired to see if there were any notable vulnerablities in the source code and the package.json by the company.

Methodology

I will follow a two-step process of attacking the site

  • Blackbox Approach: I will first perform attacks without reading the source code.
  • Whitebox Approach: Then, I will manually review the source code and document my findings. If I find any large issues, such as source code that is potentially vulnerable to XSS, I will go back and attack it.

    • Summary:

    My methodology for pen testing the system will be to first understand the source code of the ride hailing service, the see if there are any apparent vulnerabilities that I can desern from the code, and final I will use tools such a Burp Suite and curl to locate vulnerabilities. I will loop through this process until I am certain that there are no more serious security and privacy issues with the ride-hailing service.

Issues Found

  • Issue 1: CSP Scanner: Wildcard Directive
  • Location
  • Severity of issue: Medium
  • Description of issue: The following directives either allow wildcard sources, are not defined, or are overly broadly defined.
  • Proof of vulnerability. screenshots necessary
  • Resolution: Ensure that your web server, application server, load balancer, etc. is properly configured to set the Content-Security-Policy header.
  • Issue 2: X-Frame-Options Header Not Set
  • Location
  • Severity of issue: Medium
  • Description of issue: X-Frame-Options header is not included in the HTTP response to protect against 'ClickJacking' attacks.
  • Proof of vulnerability. screenshots necessary
  • Resolution: Most modern Web browsers support the X-Frame-Options HTTP header. Ensure it's set on all web pages returned by your site (if you expect the page to be framed only by pages on your server (e.g. it's part of a FRAMESET) then you'll want to use SAMEORIGIN, otherwise if you never expect the page to be framed, you should use DENY. ALLOW-FROM allows specific websites to frame the web page in supported web browsers).
  • Issue 3: Absence of Anti-CSRF Tokens
  • Location
  • Severity of issue: Low
  • Description of issue: No Anti-CSRF tokens were found in a HTML submission form. A cross-site request forgery is an attack that involves forcing a victim to send an HTTP request to a target destination without their knowledge or intent in order to perform an action as the victim. The underlying cause is application functionality using predictable URL/form actions in a repeatable way. The nature of the attack is that CSRF exploits the trust that a web site has for a user.
  • Proof of vulnerability. screenshots necessary
  • Resolution: Use a vetted library or framework that does not allow this weakness to occur or provides constructs that make this weakness easier to avoid.
